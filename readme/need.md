# 拍品信息聚合系统 - 需求文档

## 一、项目概述

输入飞书多维表格链接，自动读取拍品信息，在微信公众号搜索窑口和拍品介绍，在闲鱼搜索类似拍品参考价，所有参考资料需通过浏览器验证，只有有效链接作为来源展示。

## 二、核心需求

### 2.1 数据源

| 数据源 | 用途 | 获取方式 |
|--------|------|----------|
| 飞书多维表格 | 拍品原始数据 | 飞书 Open API 直读 |
| 微信公众号 | 拍品介绍（窑口+纹饰/器型） | 搜狗微信搜索 → 文章直链 |
| 闲鱼 | 类似拍品参考价 | 闲鱼网页版直接搜索 |
| 小茶书管理后台 | 历史拍卖成交记录 | 登录后按窑口/工艺/题材/器型搜索 |

### 2.2 飞书读取

- **输入**: 飞书多维表格链接（格式: `https://xxx.feishu.cn/base/{app_token}?table={table_id}&view={view_id}`）
- **方式**: 飞书开放平台 API（`bitable/v1`），`tenant_access_token` 认证
- **提取字段**: 拍品名字、窑口、起拍价、参考价、年代、材质、尺寸、品相、编号
- **映射规则**: 飞书字段名自动映射到标准化字段名（如"拍品名称"→name，"窑口"→kiln）

### 2.3 微信公众号搜索

#### 拍品名拆解规则

从拍品名中拆解出以下要素用于搜索和匹配:

| 要素 | 示例（宣之堂青花暗刻事事如意纹马蹄杯） | 用途 |
|------|------|------|
| 窑口 | 宣之堂 | 飞书多维表格字段，搜索必须词 |
| 工艺 | 青花暗刻 | 搜索补充词 |
| 纹饰/题材 | 事事如意 | 搜索 + 解说稿提炼 |
| 器型 | 马蹄杯 | 搜索 + 解说稿提炼 |

#### 搜索规则（强制）

1. **窑口必须匹配**: 每个搜索词必须包含窑口名称
2. **分类型搜索**: 按以下类型分别搜索，每篇文章标记 `match_type`
3. **搜索类型**:
   - `kiln_vessel`: 窑口 + 器型（如"宣之堂 马蹄杯"）
   - `kiln_motif`: 窑口 + 纹饰/题材（如"宣之堂 事事如意"）
   - `kiln_craft`: 窑口 + 工艺（如"宣之堂 青花暗刻"）
   - `kiln`: 仅窑口（兜底）

#### 解说稿提炼规则

按文章的 `match_type` 定向提炼内容:
- `kiln_vessel` → 只提炼器型描述
- `kiln_motif` → 只提炼纹饰/题材描述
- `kiln_craft` → 只提炼工艺描述
- `kiln` → 提炼通用背景
- 如果窑口+纹饰+器型都匹配 → 纹饰和器型描述都添加到解说稿

#### 展示内容

- 文章标题、摘要、来源公众号昵称
- 搜索匹配类型标签（器型匹配/纹饰匹配/工艺匹配）
- 文章详情直链（`mp.weixin.qq.com/s?src=...`）
- 拍品介绍栏右侧始终显示「手动查询」按钮

#### 文章链接要求

- **必须是文章详情链接**，不能是搜索结果列表页
- 获取方式: 搜狗微信搜索 → 同一浏览器上下文跟随跳转链 → 获取真实文章直链

### 2.4 闲鱼搜索

#### 搜索规则（强制）

1. **窑口必须匹配**: 搜索词必须以窑口为核心
2. **材质补充**: 窑口 + 材质组合搜索
3. **拍品核心词补充**: 窑口 + 拍品名简短关键词
4. **搜索结果**: 提取商品标题、价格（¥格式）、成色、地点

#### 登录要求

- 闲鱼网页版搜索需要登录 Session
- 方案: Playwright 持久化 Profile，首次扫码登录后 Cookie 保存到 `.xianyu_profile/state.json`
- Cookie 过期时自动弹出扫码窗口（60 秒内完成）

#### 商品数据获取

- 搜索页提取: 商品标题、封面图、成色
- 详情页获取: 准确售价、卖家昵称、地址
- 每件商品最多取前5个进入详情页
- 闲鱼参考价栏右侧始终显示「手动查询」按钮

### 2.5 小茶书拍卖记录搜索

#### 数据源

- 小茶书管理后台: `https://manage.funsun.cn/admin/newquotationdealinfo/`
- 登录账号: 邮箱密码登录，Session 持久化

#### 搜索规则

1. **窑口必须匹配**: `flt0_0=窑口`
2. **工艺**: `flt1_0=工艺`（从拍品名称中提取，如"青花"、"青花釉里红"）
3. **题材**: `flt2_0=题材`（从拍品名称中提取，如"梅花"、"事事如意"）
4. **器型**: `flt4_0=器型`（从拍品名称中提取，如"缸杯"、"压手杯"）

#### 输出字段

| 字段 | 来源 |
|------|------|
| 封面图 | 表格 img src |
| 拍品名称 | 表格列 |
| 起拍价 | 表格列 |
| 成交价 | 表格列 |
| 拍卖时间 | 表格列 |
| 来源 | 表格列 |
| 是否展示 | 表格列（展示/隐藏） |
| 查看原文 | 当前搜索 URL |

### 2.6 链接验证

所有参考资料链接必须满足以下条件才视为有效:

1. HTTP 状态码 200
2. 页面内容 > 50 字符（非空白页）
3. 无错误关键词（404、验证码、登录拦截、滑块验证等）
4. 页面加载超时 < 15 秒

无效链接不展示在结果中。

### 2.6 直播间解说稿

结合拍品信息和搜索结果，生成约 300 字的直播间解说稿，包含:

- 开场介绍（拍品名称、窑口、年份、尺寸、品相）
- 窑口背景知识（从微信搜索结果提取）
- 拍品亮点描述（从微信搜索结果提取）
- 市场参考信息（估价、起拍价、闲鱼同类均价）
- 竞价引导语

## 三、前端展示

### 3.1 统一结果表格

| # | 拍品名字 | 窑口 | 起拍价 | 参考价 | 状态 |
|---|----------|------|--------|--------|------|
- #: 列表中序号宽度固定适配2位数即可
- 状态列: 等待中 → 分析中 → 展开（完成后）→ 收起（展开后）
- 点击"展开"后显示参考资料和解说稿
- 再次点击"收起"折叠

### 3.2 展开内容

展开后按以下顺序展示:

1. **直播间解说稿**（红色左边框，独立卡片，按匹配类型提炼器型/纹饰/工艺描述）
2. **小茶书拍卖记录**（封面图、拍品名称、起拍价、成交价、拍卖时间、来源、查看原文）
3. **拍品介绍**（微信公众号文章列表，含标题、摘要、来源、直链、匹配类型标签）+ 右侧「手动查询」按钮
4. **闲鱼参考价**（商品列表，含标题、价格、成色、地点、直链）+ 右侧「手动查询」按钮

每个来源链接带类型标签:
- 绿色 `公众号介绍`
- 橙色 `闲鱼信息`

### 3.3 实时处理

- 使用 SSE（Server-Sent Events）流式推送处理进度
- 逐条从上到下处理拍品，每完成一条实时更新表格
- 进度条显示 X/总数

## 四、技术架构

```
action/
├── main.py                    # FastAPI 入口 + SSE 端点 + 解说稿生成
├── config.py                  # 全局配置（浏览器参数、搜索限制等）
├── login_xianyu.py            # 闲鱼独立登录脚本（首次运行）
├── requirements.txt           # Python 依赖
├── modules/
│   ├── feishu_api.py          # 飞书 Open API 集成
│   ├── feishu_reader.py       # 飞书页面 DOM 读取（文档模式）
│   ├── wechat_search.py       # 微信公众号搜索（搜狗→直链）
│   ├── xianyu_search.py       # 闲鱼搜索（登录态直搜）
│   └── link_verifier.py       # 链接有效性验证
├── utils/
│   ├── browser.py             # Playwright 浏览器管理器 + 闲鱼登录
│   └── text_parser.py         # 文本解析 + 题材/器型提取
├── templates/
│   └── index.html             # 前端页面（SSE + 展开/收起）
├── static/
│   └── style.css              # 页面样式
└── .xianyu_profile/           # 闲鱼登录态持久化（自动生成）
```

## 五、API 接口

### 5.1 分析全部拍品（SSE 流式）

```
GET /api/analyze-all?feishu_url={url}
```

事件类型:
- `start` — 返回拍品总数和摘要列表
- `progress` — 当前处理进度
- `result` — 单个拍品的完整结果（含解说稿）
- `done` — 全部处理完成
- `error` — 错误信息

### 5.2 结果数据结构

```json
{
  "index": 0,
  "name": "拍品名字",
  "kiln": "窑口",
  "starting_price": "起拍价",
  "estimate": "估价",
  "kiln_intros": [{"title": "", "snippet": "", "url": "mp.weixin.qq.com直链", "source": ""}],
  "item_intros": [{"title": "", "snippet": "", "url": "mp.weixin.qq.com直链", "source": ""}],
  "xianyu_refs": [{"title": "", "price": "", "condition": "", "url": "goofish.com直链", "location": ""}],
  "commentary": "300字解说稿"
}
```

## 六、运行方式

```bash
# 1. 安装依赖
pip install -r requirements.txt
playwright install chromium

# 2. 首次登录闲鱼（弹出浏览器扫码，60 秒内完成）
python login_xianyu.py

# 3. 启动服务
python -m uvicorn main:app --host 0.0.0.0 --port 8000

# 4. 浏览器打开
http://localhost:8000
```

## 七、配置项

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `BROWSER_HEADLESS` | True | 无头模式（登录时自动切换可见） |
| `WECHAT_SEARCH_MAX_RESULTS` | 5 | 微信每轮最大结果数 |
| `XIANYU_SEARCH_MAX_RESULTS` | 10 | 闲鱼最大结果数 |
| `LINK_VERIFY_TIMEOUT` | 15000 | 单链接验证超时（毫秒） |
| `LINK_VERIFY_CONCURRENCY` | 3 | 并发验证数 |
| `XIANYU_PROFILE_DIR` | .xianyu_profile | 闲鱼登录态存储目录 |
