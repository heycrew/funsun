# 拍卖专场成交额预估模型

## 架构

```
当前专场拍品 (30-40件)
    ├─ Stage 1: 单品落槌价预估
    │    └─ 起拍价 × 窑口×工艺溢价率 × 价格分层成交率调整
    └─ Stage 2: 专场含佣匹配
         └─ 特征向量余弦相似度 → Top N 历史场 → 加权含佣
              ↓
         动态加权融合 (20-40% S1 + 60-80% S2)
              ↓
         连续校准曲线 + 周六加成 + 特殊场检测
              ↓
         区间: best ± 37%
```

### 数据清洗
- 排除 `运营成交`（非真实成交）
- 包含 `售后`（平台退货，仍算成交）
- 竞品 `未支付` = 流拍

## Stage 1: 单品落槌价预估

### 溢价率指数

```python
# 窑口级: median(该窑口所有成交的落槌价/起拍价)
kiln_premium = {"春风祥玉": 1.657, "晓芳窑": 1.165, "天予窑": 1.733, ...}  # 86个窑口

# 窑口×工艺级: median(该组合的落槌价/起拍价)
craft_premium = {("春风祥玉","青花釉里红"): 1.856, ("春风祥玉","青花"): 1.559, ...}  # 163个组合

# 全局中位: 1.435
```

### 工艺提取
从拍品名匹配: `青花釉里红 > 釉里红 > 青花 > 珐琅彩 > 粉彩 > 矾红 > 霁蓝 > 霁红 > 郎红 > 甜白 > 紫金釉 > 娇黄釉 > 豆青釉 > 洒蓝 > 墨彩 > 斗彩`

### 计算公式
```
base = 起拍价 × 溢价率  (1元起拍 → 1000 × max(溢价率, 2.0))
adj  = base × 成交率调整系数  (1元起拍×1.20, >50K×0.90)
S1   = Σ(adj) × 1.10(佣金) × 1.10(CAL_GLOBAL)
```

### 降级
| 级 | 条件 | 溢价率来源 |
|----|------|-----------|
| 1 | 窑口×工艺 ≥5条 | craft_premium |
| 2 | 窑口 ≥5条 | kiln_premium |
| 3 | 窑口 <5条 | kiln_premium 或全局 |
| 4 | 全新窑口 | 全局中位 |

## Stage 2: 专场含佣匹配

### 特征向量 (13维)
```
[件数, 起拍总额, 均价, 中位价, P75, P90,
 春风祥玉+老贵占比(权重0.20), 窑口集中度, 窑口数量,
 竞品数, 竞品成交率, 高端标志, 周几(7位)]
```

### 匹配
- 余弦相似度 + 时间衰减(180天半衰期)
- Top 20 相似场 → 加权含佣
- 相似度 < 0.15 门控，不足3场降级全局统计

### 历史场简化
历史场只有起拍总额+件数+周几，窑口/竞品维度为0。

## 融合与校准

### 动态权重
```python
hc = 窑口×工艺匹配率  # 拍品中能匹配到craft_premium的比例
if hc > 0.7:  (S1=40%, S2=60%)
elif hc > 0.4: (S1=30%, S2=70%)
else:          (S1=20%, S2=80%)
```

### 连续校准曲线
```python
s2_anchor = max(S2_best, 50000)
tier_boost = min(2.5, 1.0 + 0.77 × (s2_anchor / 300000))
# S2=100K → ×1.26  S2=300K → ×1.77  S2=500K → ×2.28  S2=650K → ×2.5(cap)
```

### 特殊调整
- 周六: ×1.25
- 小型主题场(建盏/茶/紫砂): ×0.35
- 区间: best × (0.63 ~ 1.37)
